Amber.js

What is Amber.js?

Amber.js is a JavaScript framework for building ambitious web applications.

It provides an extensive set of architectural features that make it easy to manage the complexity of your application as it grows. Amber.js is designed to handle the tedious boilerplate that all web applications have in common, so you can focus on what's important: making your app great.

What's In It?

  • A data store that makes it easy to exchange records with a RESTful backend server.
  • Property bindings and computed properties that allow you to declare sophisticated relationships in your models that propagate changes automatically.
  • Auto-updating templates that bind to your data model and update themselves any time a change occurs.

How is Amber.js Different from Traditional Web Frameworks?

Traditional web applications force the user to download a new page every time they interact with the server. This means that every interaction is never faster than the latency between you and the user, and usually slower. Using Ajax to replace only parts of the page helps somewhat, but still requires a roundtrip to your server every time your UI needs to update. And if multiple parts of the page need to update all at once, most developers just resort to loading the page over again, since keeping everything in sync is tricky.

Amber.js, like other modern JavaScript frameworks, works a little differently. Instead of the majority of your application's logic living on the server, an Amber.js application downloads everything it needs to run in the initial page load. That means that while your user is using your app, they never have to load a new page and your UI responds quickly to their interaction.

One advantage of this architecture is that your web application uses the same REST API as your native apps or third-party clients. Back-end developers can focus on building a fast and reliable API server and don't have to be front-end experts, too.


Application Namespace


The first step in creating your application is to define a namespace for it:

        MyApp = SC.Application.create();

Creating a new instance of SC.Application does two things:

  1. Gives you a place to create your views, models and controllers.
  2. Sets up event delegation on the document so that views can handle user events (which you'll learn about later).

If your Amber application is only part of the page, you can instruct it to only listen for events that occur to children of a particular selector:

          MyApp = SC.Application.create({
            rootElement: '#sidebar'
          });

Objects

To create objects that are aware of all of the features of Amber.js, you'll create instances of SC.Object. Set properties of instances using set() and retrieve properties using get():

        var wine = SC.Object.create({
          varietal: "Merlot"
        });
        wine.get('varietal'); // "Merlot"

        wine.set('varietal', "Cabernet Sauvignon");
        wine.get('varietal'); // "Cabernet Sauvignon"

What do you get by using Amber objects? Lots, but two of the most important are:

Computed Properties

Often, your objects may have a property that is computed from one or more other properties. Instead of calling a method to ensure that these computations are up-to-date, you can define a computed property that is calculated on demand:

        var wine = SC.Object.create({
          year: 2009,
          
          age: function() {
            var wineYear = this.get('year');
            var now = new Date().getFullYear();
            
            return now - wineYear;
          }.property()
        });
        
        wine.get('age'); // 2

In this example, the age property derives its value from the year property. To have a computed property update automatically, declare the dependency like this:

Bindings

Bindings allow you to keep two properties across different objects in sync:

        MyApp.ceo = SC.Object.create({
          name: "Jeff Bezos"
        });

        MyApp.company = SC.Object.create({
          name: "Amazon",
          ceoNameBinding: 'MyApp.ceo.name'
        });
        
        MyApp.company.get('ceoName'); // "Jeff Bezos"

        MyApp.ceo.set('name', "Bill Gates");
        MyApp.company.get('ceoName'); // "Bill Gates"

Classes

You can also define reusable classes with SC.Object. Instead of using the create() method, use extend():

        MyApp.Person = SC.Object.extend({
          firstName: "John",
          lastName: "Smith",
          
          fullName: function() {
            return this.get('firstName') + ' ' + this.get('lastName');
          }.property('firstName', 'lastName')
        });
        
        var john = MyApp.Person.create();
        john.get('fullName'); // "John Smith"
        
        var steve = MyApp.Person.create({
          firstName: "Steve"
        });
        steve.get('fullName'); // "Steve Smith"

Data Store

Many web developers struggle with efficiently loading data from their server. Amber's data store makes it easy.

The data store keeps track of which models your application has already loaded, so you never make extraneous requests to the server. And it's easy to track changes to models so you know exactly which changes need to be sent back.

Create a new store like this:

        MyApp.store = DS.Store.create();

Models

Next, define each of your application's model. You can specify the type of each attribute, so it is converted into the format your application expects:

        MyApp.Person = DS.Model.extend({
          firstName: DS.attr('string'),
          lastName: DS.attr('string', {
            // Defaults to the last name to 'Smith'
            // if no value is provided by the server.
            default: 'Smith'
          }),
          
          // Use a computed property to
          // determine the fullName.
          fullName: function() {
            return this.get('firstName') + ' ' + this.get('lastName');
          }.property('firstName', 'lastName'),
          
          // Ensures that birthdays are returned as dates
          birthday: DS.attr('date')
        });

Retrieving Models

You can fetch records by ID:

        var model = MyApp.Person.find(1234);
        model.get('fullName'); // John Smith

You can also load all records of a given type:

        var people = MyApp.Person.findAll();

Associations

Models can have relationships to other models. For example, you could have blog posts that have many tags:

        MyApp.Tag = DS.Model.extend({
          name: DS.attr('string')
        });
        
        MyApp.Post = DS.Model.extend({
          title: DS.attr('string'),
          body: DS.attr('string'),
          
          tags: DS.hasMany(MyApp.Tag)
        });
        
        // Retrieve a Post object
        var post = MyApp.Post.find(1234);
        
        // Get all of the post's tags
        post.get('tags');
        // [MyApp.Tag, MyApp.Tag]
By default, the data store will talk to any RESTful JSON API. You can customize how the data store communicates with your backend by creating a subclass of DS.Adapter.
